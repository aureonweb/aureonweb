El Aula Virtual te permite compartir lecciones (videos de YouTube, archivos de Google Drive, PDF o cualquier enlace) organizadas en módulos. Tú las administras desde un panel visual con formularios: nunca necesitas editar código.

1 Las dos páginas que debes conocer

PáginaDirecciónPara qué sirve
El aula tu-sitio.com/aula.html Lo que ven tus visitantes y alumnos. También se llega desde el menú Aula Virtual.
El panel tu-sitio.com/admin.html Donde TÚ administras el contenido. No está enlazado en el sitio: escribe la dirección a mano.
Guarda admin.html en tus favoritos. Es privado: solo tú entras ahí con tu contraseña.

2 Entrar al panel por primera vez

  1. Abre tu-sitio.com/admin.html en tu navegador.
  2. La primera vez te pedirá crear una contraseña. Escríbela dos veces y pulsa Crear y entrar.
  3. Las siguientes veces solo escribirás esa contraseña para entrar.
La contraseña se guarda solo en el navegador de tu computadora. Si entras desde otra computadora, crearás una contraseña nueva para ese equipo (es normal). Si la olvidas, podrás crear otra desde el mismo equipo borrando los datos del sitio en el navegador.

3 Agregar y editar contenido

En la pestaña Contenido organizas todo en módulos y, dentro de cada módulo, lecciones.

Crear un módulo

  1. Pulsa + Agregar módulo.
  2. Haz clic en el módulo para abrirlo y llena el título (en español; el inglés es opcional) y una breve descripción.
  3. Elige una imagen de portada si quieres (una ruta como assets/v01.png o un enlace a una imagen).

Agregar lecciones

  1. Dentro del módulo, pulsa + Agregar lección.
  2. Escribe el título y la descripción.
  3. Elige el tipo y pega el enlace (URL):
TipoQué enlace pegar
YouTubeLa dirección del video, ej. https://www.youtube.com/watch?v=XXXX. El video se verá incrustado dentro del aula. La miniatura se toma sola.
Google DriveEl enlace para compartir del archivo (video, PDF, documento). En Drive: Compartir → Cualquier persona con el enlace, luego Copiar enlace.
PDFEl enlace directo a un PDF (o un PDF en Drive). Se mostrará dentro del aula.
Enlace / otroCualquier otra dirección (una carpeta de Drive, Zoom, Meet, etc.). Se abrirá en una pestaña nueva.
Para reordenar usa las flechas ▲ ▼. Para eliminar, el ícono de papelera. Todo se guarda solo mientras editas; recuerda Publicar al final.

4 Contenido público y contenido para alumnos

Cada módulo puede ser:

  • Público — cualquiera puede verlo y abrir sus lecciones.
  • Alumnos — protegido: solo se abre con el código de alumnos.

Cómo proteger un módulo

  1. En el módulo, cambia Acceso a Alumnos (con código).
  2. Ve a la pestaña Ajustes y activa “Contenido para alumnos”.
  3. Escribe el código de alumnos (la clave que compartirás con tus alumnos de paga). Ejemplo: ALUMNO2026.

¿Cómo lo usan tus alumnos?

En el aula verán los módulos de alumnos con un candado 🔒. Al pulsar Ingresar código y escribir el código correcto, el contenido se desbloquea y queda recordado en su dispositivo.

Importante sobre la protección. Los enlaces de alumnos se guardan cifrados: sin el código no se pueden abrir, ni siquiera mirando los archivos del sitio. Es una protección real. Aun así, recuerda que un alumno con el código podría compartirlo; cambia el código cada cierto tiempo (por ejemplo, cada curso) si lo necesitas. Cuando cambies el código, vuelve a Publicar: el contenido se recifra con el código nuevo y los alumnos deberán usar el nuevo.
El código por defecto del contenido de ejemplo es ALUMNO2026. Cámbialo por uno tuyo antes de abrir el aula a alumnos reales.

5 Conectar con GitHub (una sola vez)

Tu sitio vive en GitHub. Para que el botón Publicar funcione, el panel necesita un token (una llave) con permiso para actualizar el sitio. Esto se hace una sola vez.

  1. Entra a GitHub → Settings → Developer settings → Personal access tokens → Fine-grained tokens → Generate new token.
    Atajo: github.com/settings/personal-access-tokens/new
  2. En Repository access elige Only select repositories y selecciona el repositorio de tu sitio.
  3. En Permissions → Repository permissions → Contents, elige Read and write.
  4. Genera el token y cópialo (solo se muestra una vez).
  5. En el panel, pestaña Conexión, llena:
    • Usuario u organización: tu usuario de GitHub.
    • Repositorio: el nombre del repositorio (lo ves en la dirección github.com/usuario/repositorio).
    • Rama: normalmente main.
    • Ruta del archivo: déjala como data/lecciones.json.
    • Token: pega el que copiaste.
  6. Pulsa Guardar y luego Probar conexión. Debe decir “Conexión correcta”.
Estos datos se guardan solo en tu navegador, nunca se suben al sitio público.

6 Publicar los cambios

  1. (Opcional) Pulsa Vista previa para ver cómo quedará el aula antes de publicar.
  2. Pulsa Publicar y confirma.
  3. Espera ~1 minuto: GitHub actualiza el sitio solo. Recarga el aula y verás los cambios en vivo.
Si Publicar da un error, casi siempre es por la conexión: revisa el token y los datos en la pestaña Conexión y pulsa Probar conexión.

7 Respaldo: exportar e importar

  • Exportar descarga una copia de todo el contenido (archivo lecciones.json). Guárdala de vez en cuando como respaldo.
  • Importar vuelve a cargar un archivo que exportaste antes (por si necesitas restaurar).
Si algún día no quisieras usar el botón Publicar, también puedes subir el archivo lecciones.json exportado a la carpeta data/ de tu repositorio desde la web de GitHub (arrastrar y soltar en “Add file → Upload files”).

8 Seguridad y buenas prácticas

  • Usa el panel solo en tu computadora de confianza (la contraseña, el token y el código quedan guardados en ese navegador).
  • No compartas tu token de GitHub con nadie. Si se filtra, bórralo en GitHub y crea otro.
  • Comparte el código de alumnos solo con tus alumnos inscritos. Cámbialo cuando lo necesites.
  • Haz un Exportar de respaldo antes de cambios grandes.

9 Preguntas frecuentes

Publiqué pero no veo los cambios.

Espera ~1 minuto y recarga la página del aula. Si usas el navegador con caché, prueba recargar con Ctrl+F5.

El alumno dice que su código no funciona.

Verifica que sea exactamente el código actual (sin espacios) y que hayas Publicado después de definirlo. Si cambiaste el código, los alumnos deben usar el nuevo.

Quiero editar contenido de alumnos desde otra computadora.

Entra al panel, ve a Ajustes, escribe el código de alumnos y pulsa “Cargar contenido de alumnos para editar”. Así podrás ver y modificar esos enlaces.

¿Puedo tener un módulo con unas lecciones públicas y otras de alumnos?

Sí. En cada lección, el campo Acceso puede ser “Igual que el módulo”, “Público” o “Alumnos”.
